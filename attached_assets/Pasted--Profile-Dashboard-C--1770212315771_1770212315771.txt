المهمة: إعادة تصميم صفحة الملف الشخصي (Profile) بأسلوب “بطاقة جامعية + Dashboard” (تصميم C) مع دعم الوضع الليلي والنهاري + إضافة غلاف قابل للتعديل، مع الحفاظ على جميع الأدوات والوظائف الحالية.

قيود صارمة:
1) لا تغيّر أي منطق أعمال (Business logic) أو RBAC أو صلاحيات أو وظائف المتابعة/الرسائل/المنشورات/المحفوظات… إلخ. التغيير UI/Layout فقط.
2) لا تحذف أي ميزة موجودة. فقط أعد توزيعها في واجهة جديدة.
3) لا تضف قسم “الإنجازات” الآن (احذفه من الواجهة إن كان موجوداً، لكن لا تحذف كوده إن كان مستخدماً لاحقاً—فقط اخفه أو اجعل عرضه مشروطاً).
4) يجب أن تعمل الصفحة بشكل ممتاز في Dark Mode و Light Mode.

الهدف البصري (UI Spec):
A) Cover Header
- أعلى الصفحة يوجد غلاف (Cover) بعرض كامل المحتوى وارتفاع متوسط (حوالي 320–400px).
- الغلاف: صورة يرفعها المستخدم (A فقط). إذا لم توجد صورة، استخدم placeholder gradient بسيط.
- ضع overlay مناسب لضمان قراءة النص (في الداكن: overlay داكن، في النهاري: overlay خفيف).
- الصورة الشخصية (Avatar) دائرية وتظهر فوق الغلاف (floating) مع ظل خفيف، بدون إطار ذهبي/زخرفة.
- تحت الاسم مباشرة سطر إحصائي نحيف بدون أيقونات: 
  “عدد المنشورات | عدد المتابعين | عدد المتابَعين”
  مثال: “52 منشور | 181 متابع | 120 يتابع”
- أزرار: “متابعة” و “رسالة” بنفس الوظائف الحالية لكن بأسلوب Outline/Soft (لا تغيّر المنطق).

B) Dashboard Cards (بعد الغلاف)
- بعد الغلاف: شبكة كروت (Cards) أنيقة.
- على سطح المكتب: 3 أعمدة (Responsive)، وعلى الجوال عمود واحد.
- الكروت المطلوبة الآن فقط:
  1) النشاط (إحصاءات مختصرة)
  2) المنشورات (Preview + “عرض الكل”)
  3) المحفوظات (Preview + “عرض الكل”)
  4) المواد الدراسية (إن كانت موجودة حالياً في الصفحة؛ إن لم تكن موجودة، لا تخترع بيانات — فقط لا تضع الكرت)
- لا تعرض “الإنجازات” الآن.

C) Cover Editing (UI + Storage)
- أضف زر صغير على الغلاف يظهر فقط لصاحب الحساب/الأدمن: “تعديل الغلاف” (أو أيقونة قلم).
- عند الضغط: اختيار صورة (file input) + معاينة + حفظ.
- تخزين الغلاف: استخدم نفس نمط التخزين الموجود بالمشروع (localStorage / replaceable storage pattern IKeyValueStorage إن كان معمول).
- key مقترح: daam_profile_cover_v1 (مربوط بـ userId)
- خزن فقط Data URL (base64) مؤقتاً. لا تغيّر الباك-إند.
- وفر زر “إزالة الغلاف” يرجّع للـ placeholder.

D) Dark/Light Mode
- التزم بتوكنز الثيم الحالية بالمشروع (لا تدخل ألوان hardcoded إلا للضرورة القصوى).
- الكروت في الداكن: خلفية أغمق من الصفحة بدرجة واحدة، مع borders خفيفة.
- في النهاري: خلفية عاجية/فاتحة وكروت بيضاء بظل خفيف.
- لا تغيّر منطق زر تبديل الوضع إن موجود.

خطوات التنفيذ المطلوبة:
1) اعثر على صفحة الملف الشخصي:
   - ابحث عن route مثل /profile أو /u/:id أو /user/:id أو كلمة Profile في pages.
   - حدد الملف/المكوّن المسؤول (مثلاً client/src/pages/... أو client/src/components/...).
2) افصل أجزاء UI: CoverHeader + StatsLine + ActionButtons + CardsGrid.
3) أعد ترتيب العناصر حسب الـ UI Spec أعلاه.
4) اربط الإحصائيات (posts/followers/following) بالبيانات الحالية (لا تنشئ بيانات وهمية).
5) نفّذ Cover editing كما وُصف، مع صلاحية التحرير لصاحب الحساب.
6) تأكد أن التنقل وفتح المنشورات والمحفوظات وملف المستخدم وكل الأدوات تعمل مثل قبل.
7) أضف فحص بصري سريع:
   - Desktop + Mobile
   - Dark + Light
8) اكتب ملخصاً نهائياً: الملفات المعدلة + ما الذي تغير + كيف أختبر.

معايير القبول (Acceptance Criteria):
- الشكل أصبح “بطاقة جامعية + Dashboard” وليس مثل تويتر.
- سطر الإحصائيات يظهر بوضوح تحت الاسم (منشورات/متابعين/يتابع).
- لا يوجد قسم إنجازات في الواجهة.
- الغلاف قابل للتعديل والحفظ والإزالة ويعمل في الوضعين.
- لا توجد كسور في الوظائف (متابعة/رسالة/منشورات/محفوظات/روابط).
