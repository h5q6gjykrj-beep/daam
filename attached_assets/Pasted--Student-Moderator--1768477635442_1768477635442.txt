نظام التسجيل والصلاحيات (Student / Moderator) – النسخة المصححة

الهدف

نريد نظام تسجيل يعتمد على تأكيد البريد الإلكتروني عبر رابط (Email Verification Link)، مع فصل واضح بين:
	•	الطالب
	•	المشرف
ودون استخدام رموز تحقق رقمية (OTP).

⸻

1) الطالب (Student)

شروط التسجيل
	•	يُسمح بالتسجيل فقط باستخدام البريد الجامعي:
	•	مثال: @utas.edu.om
	•	أي بريد لا ينتهي بالنطاق الجامعي:
	•	يتم رفض التسجيل مباشرة

بيانات التسجيل المطلوبة

عند تسجيل الطالب يُطلب إدخال:
	1.	البريد الجامعي
	2.	كلمة المرور
	3.	رقم الهاتف
	4.	المحافظة والولاية

التحقق
	•	بعد التسجيل يتم إرسال رسالة تأكيد إلى البريد الجامعي
	•	تحتوي الرسالة على رابط تأكيد
	•	لا يتم تفعيل الحساب إلا بعد الضغط على الرابط

⸻

2) المشرف (Moderator)

طريقة التسجيل
	•	المشرف يسجل من نفس صفحة تسجيل الطلاب
	•	ولكن باستخدام بريد إلكتروني خاص واحد محدد مسبقًا (Hotmail)
	•	مثال: example@hotmail.com
	•	لا يُسمح باستخدام أي بريد Hotmail آخر

التحقق (مثل الطالب ولكن بشرط إضافي)
	•	بعد التسجيل يتم إرسال رسالة تأكيد إلى بريد المشرف
	•	تحتوي على رابط تأكيد
	•	بعد الضغط على الرابط:
	•	يتم تفعيل الحساب
	•	يتم تعيين الدور تلقائيًا: role = moderator

لا يتم استخدام رمز رقمي
التحقق يتم فقط عبر رابط التأكيد في البريد

⸻

3) صلاحيات المشرف (داخل الموقع)

بعد تفعيل الحساب، يمتلك المشرف الصلاحيات التالية داخل الموقع:
	•	حذف منشور أو تعليق
	•	تعديل منشور أو تعليق
	•	حظر مستخدم
	•	طرد / إيقاف مستخدم
	•	تدريب المساعد الذكي (في النسخة الحالية)

ويظهر بجانب اسمه داخل:
	•	المنشورات
	•	الملف الشخصي
شارة واضحة: مشرف

⸻

4) تذكرني + الدخول بالبصمة/الوجه (لكل المستخدمين)

تذكرني (Remember Me)
	•	خيار “تذكرني” في صفحة تسجيل الدخول
	•	عند تفعيله:
	•	يبقى المستخدم مسجّلًا بدون إعادة إدخال كلمة المرور

البصمة / الوجه
	•	دعم الدخول السريع بالبصمة أو Face ID عند استخدام الهاتف
	•	متاح لكل:
	•	الطلاب
	•	المشرف
	•	يعتمد على دعم الجهاز والمتصفح (مثل WebAuthn / Passkeys)
	•	في حال عدم الدعم:
	•	يتم الرجوع تلقائيًا لتسجيل الدخول العادي

⸻

5) الخصوصية

ما يظهر للعامة
	•	الاسم
	•	الصورة الشخصية والغلاف
	•	التخصص
	•	عدد المتابعين / يتابع / المنشورات
	•	منشوراته وردوده

ما لا يظهر للعامة
	•	البريد الإلكتروني
	•	رقم الهاتف
	•	المحافظة والولاية

هذه البيانات:
	•	تظهر فقط لصاحب الحساب داخل الإعدادات
	•	ولا تظهر لأي مستخدم آخر

⸻

6) ملاحظات أمنية
	•	البريد الخاص بالمشرف يجب حفظه في:
	•	إعدادات النظام (Config)
	•	أو Environment Variables
	•	لا يُكتب في الواجهة أو الكود الظاهر
	•	رابط التأكيد:
	•	صالح لفترة محدودة
	•	يُستخدم مرة واحدة فقط

	•	يتم رفضه مباشرة

ملاحظة أمنية

	•	إعدادات النظام (Config)
