مهمتك: إضافة "بحث ذكي" داخل شريط التحكم sticky في صفحة /feed يقوم بفلترة المنشورات بشكل متسامح مع اختلافات الكتابة العربية.

⚠️ قيود صارمة:
- عدّل ملف واحد فقط: client/src/pages/feed.tsx
- لا تعدّل أي ملفات أخرى
- لا تغيّر API أو منطق الجلب
- التغيير: UI + فلترة محلية على البيانات المعروضة فقط

المطلوب UI:
1) أضف حقل بحث (Input) داخل الشريط sticky (قرب sort/chips) مع أيقونة بحث.
- placeholder بالعربي: "ابحث عن جامعة، كلية، أو موضوع…"
- بالإنجليزية: "Search university, college, or topic…"
- يدعم RTL/LTR

المطلوب Logic:
2) أنشئ دالة تطبيع نص عربي/عام داخل feed.tsx (بدون مكتبات جديدة) مثل:
normalizeText(s: string): string
وتقوم بالآتي:
- lowerCase
- remove diacritics (التشكيل)
- توحيد الحروف:
  إ/أ/آ => ا
  ة => ه
  ى => ي
  ؤ => و
  ئ => ي
- إزالة "ال" التعريف في بداية الكلمات قدر الإمكان:
  - إزالة "ال" عندما تأتي كبادئة لكلمة (مثلاً "الهندسة" تصبح "هندسه")
  - لا تزيل "ال" داخل الكلمة
- إزالة المسافات المكررة
- يمكن إزالة الرموز غير الحروف/الأرقام إن لزم (اختياري)

3) فلترة المنشورات:
- أضف state: const [searchQuery, setSearchQuery] = useState("")
- طبّق الفلترة على القائمة النهائية المعروضة (بعد فلترة التصنيفات/الترتيب إن وجدت):
  - إذا searchQuery فارغ: اعرض كل شيء
  - إذا غير فارغ:
    - كوّن نص قابل للبحث لكل منشور بجمع:
      العنوان/النص + اسم المستخدم + الجامعة/الكلية/التخصص + الوسوم (حسب الحقول المتاحة في بيانات post)
    - طبّع (normalizeText) لكلٍ من query و postSearchText
    - اعرض المنشور إذا normalizedPostSearchText includes normalizedQuery

4) تجربة الاستخدام:
- بمجرد كتابة المستخدم يتم الفلترة مباشرة (onChange).
- أضف زر صغير داخل الـInput لمسح البحث (X) يظهر فقط عند وجود نص.
- أضف chip صغيرة توضح البحث الحالي: "🔍 {query}" (اختياري) مع زر X لمسحها.

معايير نجاح:
- كتابة "ادارة" تطابق منشورات فيها "إدارة"
- كتابة "هندسه" تطابق "الهندسة" و"هندسة"
- كتابة بدون "ال" تطابق مع "ال"
- لا يتم تعديل أي ملف غير feed.tsx
- لا يتأثر باقي التصميم الحالي

بعد التنفيذ:
- اذكر أين أضفت normalizeText
- واذكر أين طبقت filter على posts
- واذكر أين أضفت حقل البحث في JSX
